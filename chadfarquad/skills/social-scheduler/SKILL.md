# Social Scheduler — Content Calendar & Post Planning

Help the user plan, organize, and draft social media content. Chad creates the plan and writes the posts — the user handles publishing (or uses their existing scheduling tool).

## Process

### 1. Understand Their Social Presence

Check MEMORY.md for:
- Which platforms they're active on (LinkedIn, Instagram, Pinterest, etc.)
- How often they post
- What type of content performs well
- Their brand voice and visual style

If this isn't documented yet, ask:
- "Which platforms do you post on?"
- "How often do you want to post?" (daily, 3x/week, weekly)
- "What type of content resonates with your audience?"

### 2. Create a Content Calendar

Build a weekly or monthly content plan:

```markdown
# Content Calendar — Week of [Date]

## Monday
- **Platform:** LinkedIn
- **Type:** Industry insight
- **Topic:** [Topic]
- **Status:** Draft ready

## Wednesday
- **Platform:** Instagram
- **Type:** Behind the scenes
- **Topic:** [Topic]
- **Status:** Needs draft

## Friday
- **Platform:** LinkedIn + Instagram
- **Type:** Value post
- **Topic:** [Topic]
- **Status:** Idea only
```

Save to `memory/drafts/content-calendar-YYYY-MM-DD.md`

### 3. Draft Posts

For each slot in the calendar, write the full post:

**LinkedIn:**
- Hook line (first line that shows before "see more")
- Body (3-5 short paragraphs, lots of white space)
- CTA or question to drive engagement
- Hashtags (3-5 relevant ones)

**Instagram:**
- Caption with hook
- Body text
- Hashtags (up to 30, mix of broad and niche)
- Alt text suggestion for the image

**Pinterest:**
- Pin title (keyword-rich)
- Pin description
- Board suggestion

### 4. Present for Review

```
📅 Content calendar — this week:

Mon: LinkedIn — [topic] ✅ Draft ready
Wed: Instagram — [topic] ✅ Draft ready
Fri: LinkedIn + IG — [topic] 📝 Needs your input

Want to review the drafts?
```

### 5. Track What's Published

When the user confirms they've posted:
- Move from "Draft ready" to "Published" with date
- Note any early engagement metrics if shared
- Use this data to inform future content planning

## Content Themes

Suggest a rotating set of content themes based on the user's business:

| Day | Theme | Example |
|---|---|---|
| Monday | Industry insight | Trends, news, hot takes |
| Tuesday | Behind the scenes | Process, workspace, day-in-the-life |
| Wednesday | Value / education | Tips, how-tos, frameworks |
| Thursday | Social proof | Client wins, testimonials, results |
| Friday | Personal / human | Story, reflection, weekend plans |

Adapt themes to the user's brand and audience.

## Rules

- Write platform-native content. LinkedIn posts don't read like Instagram captions.
- Don't suggest posting more than the user can sustain. Consistency > volume.
- Save all drafts to memory/drafts/ — they're reusable.
- If the user shares what performed well, note the pattern in MEMORY.md.
- Never post on behalf of the user unless they have an automated publishing tool AND explicitly confirm.
